# Dataset, Model and JSON Guide

This guide explains the processed data format, dataset generation, PyTorch model format and online JSON output.

## 1. Core representation

The Python pipeline extracts:

```text
dataSSB   = 360 x 6
rxGridSSB = 240 x 4
```

For `N` samples:

```text
dataSSB   = 360 x 6 x N
rxGridSSB = 240 x 4 x N
```

The recut from `dataSSB` to `rxGridSSB` follows the MATLAB convention:

```matlab
rxGridSSB = dataSSB(61:300, 2:5)
```

Python indexing:

```python
rxGridSSB = dataSSB[60:300, 1:5]
```

## 3. Capture a new Python dataset

```bash
cd ~/AlbertoDir/DT_sensing_fusion
source .venv_uhd/bin/activate

python src/python/ssb_python/capture_online_rxgridssb_dataset_cfo.py \
  --serial 34B73C3 \
  --freq 3541.44e6 \
  --rate 15.36e6 \
  --gain 60 \
  --duration-ms 20 \
  --num-iters 200 \
  --warmup-iters 10 \
  --channel 0 \
  --force-nid2 0 \
  --enable-cfo-correction \
  --cfo-warmup-iters 30 \
  --cfo-correction-sign -1 \
  --progress-every 10
```

Output:

```text
results/python_online_rxgridssb_dataset_cfo/
```

These H5 files are generated data and should not be committed.

## 5. Model checkpoint format

Current checkpoint:

```text
results/binary_empty_vs_P5_rx/model_rxGridSSB/model.pt
```

It is loaded with:

```python
torch.load(path, map_location="cpu", weights_only=False)
```

Expected contents:

```text
model_state_dict
mean
std
model_name
input_shape
complex_mode
classes
config
```

Current expected values:

```text
input_shape = [2, 240, 4]
complex_mode = abs_phase
classes = ["empty", "P5"]
```

Input tensor:

```text
[batch, 2, 240, 4]
```

Channels:

```text
channel 0 = abs(rxGridSSB)
channel 1 = angle(rxGridSSB)
```

Normalization:

```text
x = (x - mean) / std
```

where `mean` and `std` are stored in the checkpoint.

## 7. Use a different model

The online script supports:

```bash
--inference-backend torch
--torch-model path/to/model.pt
```

Example:

```bash
python src/python/ssb_python/online_5g_python_cfo_json_scp.py \
  ... \
  --inference-backend torch \
  --torch-model results/binary_empty_vs_P5_rx/model_rxGridSSB/model.pt
```

A replacement checkpoint should currently use the same model interface:

```text
input_shape = [2, 240, 4]
complex_mode = abs_phase
binary output logit
```

If the architecture changes, update:

```text
src/python/ssb_python/rxgrid_torch_inference.py
```

## 9. Training a new model

Historical training script:

```text
src/python/train_datassb_binary_pipeline.py
```

Recommended future training flow:

```text
1. Capture EMPTY samples with the Python CFO pipeline.
2. Capture P5/person samples with the Python CFO pipeline.
3. Store labels and H5 paths.
4. Build tensors:
   channel 0 = abs(rxGridSSB)
   channel 1 = angle(rxGridSSB)
5. Train a binary classifier.
6. Save checkpoint with:
   model_state_dict
   mean
   std
   input_shape = [2, 240, 4]
   complex_mode = abs_phase
   classes
   config
7. Deploy with:
   --torch-model path/to/new/model.pt
```

The preferred practice is to train and infer with the same Python extraction pipeline.
